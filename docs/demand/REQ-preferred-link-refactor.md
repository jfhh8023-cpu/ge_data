# 需求文档 — 首选收集项 & 系统级链接重构

**版本**: v1.1 已确认  
**日期**: 2026-04-17  
**状态**: ✅ 已确认，可进入开发

---

## 一、需求背景

当前系统中，每个任务单独管理一套填写链接（任务维度），导致：
- 每次新建任务都要重新生成/分发链接
- 人员页只展示名单，没有便捷入口
- 无法快速切换当前最常用的任务作为"默认填写项"

本次重构将链接升级为系统级别（人员维度），并引入「首选收集项」概念，提升整体使用效率。

---

## 二、功能需求

### F1 — 首选收集项（任务收集列表）

#### F1.1 入口
- 任务收集列表，每行操作列新增按钮，位置在「停止收集」前
- 初始文字：**「设置为首选收集项」**
- 设置后文字变为：**「取消收集首选」**

#### F1.2 唯一性约束
- 全系统同一时刻**仅允许一个任务**处于首选状态（显示「取消收集首选」）
- **仅允许状态为「收集中（active）」的任务被设为首选**；closed 任务操作列不展示该按钮

#### F1.3 设置流程
```
点击「设置为首选收集项」（仅 active 任务可见此按钮）
    ├── 当前无首选任务 → 直接设置，按钮变「取消收集首选」
    └── 已有其他任务为首选
            ↓
        弹出确认框：
        「《{已设首选任务标题}》已开启收集首选，是否切换为当前任务《{当前任务标题}》作为首选收集项？」
            ├── 确认 → 旧任务取消首选 → 新任务设为首选
            └── 取消 → 不操作
```

#### F1.4 取消流程
- 点击「取消收集首选」→ 直接取消，无需弹框，按钮变回「设置为首选收集项」

### F2 — 团队人员列表展示

#### F2.1 展示方式
- 团队人员页（PersonnelPage）改为**列表（el-table）形式**展示人员
- 列：姓名 | 角色 | 在职状态 | 专属链接操作区

#### F2.2 链接操作区（每人一行，位于最右侧）

每行链接操作区包含以下元素：

1. **预设文本框**  
   - 可编辑，默认内容：`请填写上周工作内容，您的专属链接如下：`
   - 宽度自适应，支持手动修改

2. **「打开链接」**（可点击文字/按钮）  
   - 点击在新标签页打开该人员的专属填写页

3. **「复制链接」**  
   - 仅复制纯链接 URL，不含其他内容

4. **「复制带标题的链接」**  
   - 复制内容格式：`{姓名}同学{预设文本框内容}\n{链接URL}`  
   - 示例：`朱俊锋同学请填写上周工作内容，您的专属链接如下：\nhttp://...`

### F3 — 系统级专属链接

#### F3.1 链接语义变更

| 旧方案 | 新方案 |
|--------|--------|
| 每个任务×每个人员生成一条链接 | 每个人员有且仅有一条永久链接 |
| token 标识「某任务的某人」 | token 标识「某人」 |
| 链接随任务变化 | 链接固定，跨任务复用 |

#### F3.2 链接位置变更
- **移除** TaskDetail.vue 中的「链接管理」Tab（迁移完成后同步移除）
- 链接统一展示在团队人员列表每行（F2）

#### F3.3 链接生成规则
- 新增人员时自动生成系统级 token，规则：`UUID` 或 `staffId_时间戳`
- 每人唯一，不可重复生成（幂等）
- 现有人员通过迁移脚本一次性生成

#### F3.4 数据迁移说明
- 现有 `fill_links` 表中，同一 `staff_id` 可能有多条（不同 task）
- 迁移策略：取每个 `staff_id` 最新一条 `fill_links` 的 token，作为该人的系统级 token
- 无历史 fill_link 的人员自动生成新 token
- 旧 `fill_links` 表保留一段时间后废弃，链接管理 Tab 迁移完成后一并移除

### F4 — 填写页变更

#### F4.1 默认任务展示
- 打开专属链接，默认展示**「首选收集项」**对应的任务内容（左侧表单可填写）
- **若当前无首选任务**：
  - 左侧显示提示语：「暂无开放中的收集任务，请联系管理员」
  - **右栏「我的历史」正常展示**，历史编辑功能完整可用（点击历史任务编辑按钮，左侧照常展开编辑）

#### F4.2 我的历史 — 编辑功能

**位置**：填写页右栏「我的历史」，每个任务条目的「收集中 / 已停止」标签右侧新增**「编辑」按钮**

**触发流程**：
```
点击「编辑」按钮
    ↓
左侧表单区切换为该任务的已提交 work_records 内容（逐行回显）
左侧标题前追加标记：【编辑历史数据】
左侧顶部（若当前系统有首选任务）显示按钮：「← 返回最新工时收集」
右下角提交按钮文字改为：「编辑完成」
    ↓
用户修改后点击「编辑完成」
    ├── 任务状态为 active（收集中）→ 保存记录 + 全系统广播数据变更
    └── 任务状态为 closed（已停止）→ 提示「该任务已停止收集，无法修改」，不保存
```

**「← 返回最新工时收集」按钮**：
- 显示条件：当前处于历史编辑模式 AND 系统存在首选任务
- 点击后：退出历史编辑模式，左侧恢复展示首选任务填写界面
