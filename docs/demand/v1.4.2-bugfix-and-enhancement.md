# DevTracker v1.4.2 — 需求文档

> 日期：2026-04-17
> 来源：用户反馈

## 需求列表

### BUG-1：周期统计-部门全观明细表列错位
- **现象**：产品经理列显示的是工时数据，工时列为空
- **影响页面**：周期统计 → 部门全观 → 下方明细表

### BUG-2：周期统计-个人聚焦-单独查看列错位
- **现象**：与 BUG-1 相同，产品经理列显示工时，工时列为空
- **影响页面**：周期统计 → 个人聚焦 → 单独查看 → 展开任务后的表格

### BUG-3：需求工时统计-本周/上周标题显示有误
- **现象**：第15周应该显示"上周"，但标题前缀不对
- **期望行为**：
  - 若存在包含当前日期的统计任务 → 标题显示"本周-第XX周"，默认选中
  - 若不存在 → 标题显示"上周-第XX周"，默认选中上一周任务
  - XX 为 ISO 日历维度的周数

### UI-1：概要卡片排序调整
- **当前**：总工时 → 记录数 → 人员 → 任务数 → 前端 → 后端 → 测试
- **期望**：总工时 → 前端 → 后端 → 测试 → 记录数 → 人员 → 任务数

### UI-2：明细表总计行格式
- **期望**：`合计     xxx.xx小时`（去掉"一共"）

### UI-3：填写工时页面优化
- 左右空白再减半
- 暂存草稿/提交按钮要大
- 右上角收集状态要明确显示（已停止收集 / 正在收集中）

### FEAT-1：需求工时统计-编辑模式不可编辑行提示
- 编辑模式下，不可编辑的行操作列显示"不可编辑"
