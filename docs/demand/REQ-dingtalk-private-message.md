# 需求文档：钉钉私聊消息发送

> 文档编号：REQ-DINGTALK
> 日期：2026-04-17
> 版本：v1.0
> 状态：已确认

---

## 1. 需求背景

DevTracker 工时统计系统中，管理员在「任务详情 → 链接管理」页面可以看到每位员工的专属填写链接。目前「发送」按钮仅为占位功能（弹出提示），需要实现真正的钉钉消息发送能力，让管理员一键将填写链接通过钉钉私聊发送给对应员工。

## 2. 需求目标

- 管理员点击「发送」按钮后，系统自动通过钉钉网页版将预设消息（含专属链接）发送给目标员工的钉钉私聊
- 全过程后台运行，不弹出窗口，不抢占鼠标键盘，用户无感
- 员工在钉钉中收到一对一私聊消息，点击链接即可跳转到填写页

## 3. 用户角色

| 角色 | 操作 |
|------|------|
| 管理员 | 在链接管理页点击「发送」；首次配置时扫码登录钉钉网页版 |
| 员工 | 在钉钉中收到私聊消息，点击链接填写工时 |

## 4. 功能需求

### 4.1 发送钉钉私聊消息

**触发入口**：任务详情 → 链接管理 Tab → 操作列 → 点击「发送」按钮

**前置条件**：
- 系统已完成钉钉网页版扫码登录（会话有效）
- 该链接关联了有效的员工姓名

**主流程**：
1. 管理员点击某一行的「发送」按钮
2. 按钮变为加载状态，显示「发送中...」
3. 系统后台自动：
   - 启动 headless 浏览器
   - 加载钉钉网页版会话
   - 搜索目标员工姓名
   - 进入一对一聊天窗口
   - 输入预设消息并发送
4. 发送成功：按钮恢复，弹出「已发送给 xxx」
5. 发送失败：弹出错误提示

**消息内容模板**：
```
【工时填写提醒】
{任务标题}
请点击链接填写：{专属填写链接URL}
```

**示例**：
```
【工时填写提醒】
语音业务线-2026年第16周工时统计
请点击链接填写：http://jfzhu8023.cloud/devtracker/fill/abc123
```

### 4.2 钉钉登录管理

**触发入口**：后端 API 调用（管理员操作）

**首次登录流程**：
1. 管理员调用登录接口 `POST /api/dingtalk/login`
2. 系统弹出浏览器窗口，打开钉钉网页版登录页
3. 管理员用手机钉钉扫描二维码
4. 扫码成功后，系统自动保存会话状态并关闭浏览器
5. 返回「登录成功」

**登录状态查询**：
- `GET /api/dingtalk/status` 返回当前是否已登录

**会话过期处理**：
- Cookie 有效期约 30 天
- 发送消息时自动检测会话是否过期
- 过期时前端提示「钉钉会话已过期，请联系管理员重新扫码登录」
- 管理员重新执行登录流程即可

### 4.3 批量发送（二期，本次不实现）

- 「一键发送全部」按钮，遍历所有未提交员工逐一发送
- 发送进度条展示
- 跳过已提交的员工

## 5. 非功能需求

| 项目 | 要求 |
|------|------|
| 性能 | 单条消息发送耗时 ≤ 8 秒 |
| 并发 | 同一时间只允许一条发送任务（队列串行） |
| 安全 | 会话文件不得提交到 Git，需加入 .gitignore |
| 兼容 | 后端服务器需支持运行 Chromium（Linux/Windows 均可） |
| 可维护性 | 钉钉网页版 DOM 变更时，仅需更新选择器配置 |

## 6. 技术方案

采用 **Playwright + 钉钉网页版 (im.dingtalk.com)** 方案：
- 后端 Node.js 集成 Playwright
- 通过 `storageState` 保存/复用钉钉登录会话
- headless Chromium 后台运行，用户无感
- 详见开发文档 `DEV-dingtalk-private-message.md`

## 7. 验收标准

- [ ] 首次扫码登录成功，会话文件生成
- [ ] 点击「发送」后，目标员工钉钉收到私聊消息
- [ ] 消息内容包含正确的任务标题和专属链接
- [ ] 发送过程中不弹出任何窗口，不影响用户操作
- [ ] 会话过期时有明确的错误提示
- [ ] 重新扫码后可恢复发送功能
- [ ] 会话文件已被 .gitignore 排除
