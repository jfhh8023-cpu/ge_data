## 3. REQ-39 自动任务执行模式选择

### 3.1 入口位置

在设置页每条自动任务规则的单行规则中，在“时间选择框”和“保存规则”按钮之间新增下拉选项。

### 3.2 可选值

下拉选项包含三项：

- `执行规则并通知`
- `仅执行规则`
- `仅通知`

后端存储建议值：

- `run_and_notify`
- `run_only`
- `notify_only`

### 3.3 默认值与兼容

- 新建规则默认选择 `执行规则并通知`。
- 旧规则没有执行模式字段时，后端和前端均按 `执行规则并通知` 处理。

### 3.4 执行规则并通知

当模式为 `执行规则并通知`：

- 主启用开关关闭：该规则自动触发时不执行、不通知。
- 主启用开关打开，钉钉 webhook 通知开关打开：保持当前逻辑，自动新增任务，新增成功后发送 webhook 通知。
- 主启用开关打开，钉钉 webhook 通知开关关闭：仅自动新增任务，不发送 webhook 通知。

### 3.5 仅执行规则

当模式为 `仅执行规则`：

- 钉钉 webhook 通知开关强制为关闭。
- 钉钉 webhook 通知开关不可打开。
- 鼠标悬浮在通知开关区域时提示：`当前任务仅新增任务，已禁止开放通知`。
- 自动触发时仅新增任务，不发送 webhook 通知。
- 测试执行按钮可用，含义为立即测试新增任务，不发送 webhook 通知。

### 3.6 仅通知

当模式为 `仅通知`：

- 钉钉 webhook 通知开关强制为打开。
- 钉钉 webhook 通知开关不可关闭。
- 鼠标悬浮在通知开关区域时提示：`当前任务仅通知，已禁止生成任务`。
- 自动触发时不新增任务，仅发送 webhook 通知。
- 测试执行按钮置灰不可点击。
- 鼠标悬浮在测试执行按钮上提示：`当前任务仅通知，已禁止生成任务`。

## 4. REQ-40 钉钉 webhook 通知子模块视觉层级

### 4.1 目标

钉钉 webhook 通知模块应表现为当前规则行的子模块，而不是与规则行同级的独立区域。

### 4.2 页面要求

- 在规则行下方绘制一根子目录样式的直角连接线。
- 钉钉 webhook 通知模块整体向右缩进。
- 保持现有通知配置、机器人列表、通知内容、接收人和测试发送能力不丢失。
- 不新增大面积卡片嵌套，避免影响设置页整体密度。

---

## 5. REQ-41 通知操作按钮顺序与文案调整

### 5.1 按钮顺序

钉钉 webhook 通知模块中的操作按钮调整为：

1. `保存通知`
2. `配置接收人`
3. `测试发送webhook`

### 5.2 文案调整

- 原 `接收人` 改为 `配置接收人`。
- 原 `测试发送` 改为 `测试发送webhook`。

## 6. 后端开发方案

1. `AutoTaskRule` 新增字段 `action_mode`，默认 `run_and_notify`。
2. 启动时通过兼容迁移确保 `auto_task_rules.action_mode` 存在。
3. `normalizeRulePayload` 校验并保存 `action_mode`。
4. 保存规则时根据模式强制归一化通知开关：
   - `run_only`：`notify_enabled = false`
   - `notify_only`：`notify_enabled = true`
   - `run_and_notify`：保留用户配置
5. `runRuleOnce`：
   - `notify_only`：禁止测试执行新增任务，返回明确错误。
   - `run_only`：仅新增任务，不通知。
   - `run_and_notify`：保持当前逻辑。
6. 自动调度 `executeRule`：
   - `notify_only`：只发送 webhook 通知，不调用新增任务逻辑。
   - `run_only`：只新增任务，不发送 webhook 通知。
   - `run_and_notify`：保持当前逻辑。

---

## 7. 前端开发方案

1. 设置页规则行新增执行模式下拉框。
2. 切换执行模式时立即同步通知开关状态：
   - `仅执行规则`：关闭通知开关。
   - `仅通知`：打开通知开关。
3. 通知开关按执行模式禁用，并显示对应 tooltip。
4. `仅通知` 时禁用“测试执行”按钮，并显示对应 tooltip。
5. 钉钉 webhook 通知区域增加子目录直角连接线与右缩进样式。
6. 调整通知操作按钮顺序和文案。

## 8. 验收标准

- 规则行在时间选择框和保存规则之间出现执行模式下拉框。
- 旧规则默认显示为 `执行规则并通知`。
- `仅执行规则` 模式下通知开关关闭且不可打开，悬浮提示正确。
- `仅通知` 模式下通知开关打开且不可关闭，悬浮提示正确。
- `仅通知` 模式下测试执行按钮置灰不可点击，悬浮提示正确。
- 钉钉 webhook 通知模块整体向右缩进，并有直角子目录连接线。
- 通知按钮顺序为 `保存通知`、`配置接收人`、`测试发送webhook`。
- 后端语法校验通过。
- 前端生产构建通过。
