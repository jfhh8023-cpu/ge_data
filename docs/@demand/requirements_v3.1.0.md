## 1. 本次需求铁律

### 1.1 修改范围铁律

本次开发只允许修改本需求文档明确提及的范围，不允许顺手重构、顺手改样式、顺手改历史逻辑。

允许触达的范围仅包括：

- 设置入口与设置页面。
- 自动生成下一个任务收集任务。
- 自动任务触发后的钉钉群机器人 webhook 通知。
- 需求工时统计的全量周期数据备份下载。
- 系统内已有年份下拉的“当前实际年份 + 1”显示能力。
- 底部信息置底与底部文案。
- 为实现上述功能必须新增的后端接口、数据表、权限资源和调度服务。

### 1.2 历史功能保护铁律

不得影响历史已有功能，包括但不限于：

- 任务收集列表、任务详情、创建任务、编辑任务、删除任务、停止任务、开启任务。
- 首选任务、系统级填报链接、填报历史、人员调入调出。

- 周期统计页面现有统计、人员维度、导出能力。
- 人员管理、权限控制、访问链接权限。
- 现有 `GET /api/tasks`、`POST /api/tasks`、`/api/report`、`/api/stats`、`/api/fill`、`/api/permissions` 等接口的既有行为。

### 1.3 钉钉能力边界铁律

本次钉钉通知仅使用钉钉群机器人 webhook。

不得恢复或新增以下能力：

- 不使用 Playwright 自动登录钉钉网页端。
- 不使用钉钉私聊自动发送。
- 不使用桌面客户端自动化。
- 不新增需要人工扫码、人工登录、浏览器会话保持的通知链路。

## 5. REQ-31 自动执行任务并通知

### 5.1 用户故事

作为管理员，我希望系统可以按我配置的时间自动生成下一个任务收集任务；生成成功后，如果通知开关开启，则自动向指定钉钉群机器人 webhook 推送预设内容。

### 5.2 模块名称

设置页面新增模块：

`自动执行任务并通知`

### 5.3 自动任务配置能力

系统支持配置单条或多条自动任务规则。

每条规则包含：

- 规则名称。
- 启用状态。
- 调度类型。
- 调度日期。
- 调度时间。
- 通知开关。
- 钉钉 webhook 机器人配置。
- 钉钉 webhook 发送内容。
- 最近一次执行结果。

### 5.4 调度类型

#### 5.4.1 按年内每月日期执行

管理员可以配置：

- 指定某一年。
- 在该年内每个月的某一天或多个日期执行。
- 精确到几点、几分、几秒。

示例：

- 2026 年每月 1 日 09:00:00 执行。
- 2026 年每月 1 日、15 日 09:00:00 执行。

边界规则：

- 若某月不存在配置的日期，例如 2 月没有 30 日，则该月跳过该日期。
- 到达配置年份之外后，该规则不再触发，除非管理员重新编辑年份。

#### 5.4.2 按每周星期执行

管理员可以配置：

- 每周星期几执行。
- 可选择一个或多个星期。
- 精确到几点、几分、几秒。

示例：

- 每周一 09:00:00 执行。
- 每周一、周三、周五 18:30:00 执行。

### 5.5 时间基准

所有自动任务触发、周数判断、日期判断均以北京时间为准。

北京时间定义：

- 时区：`Asia/Shanghai`
- 日期：北京时间自然日。
- 周数：北京时间日期对应的 ISO 周数。

### 5.6 自动生成任务规则

当某条自动任务规则到达配置时间时，系统执行以下判断：

1. 查询当前系统内已有的周维度任务收集任务。
2. 找到当前系统内最新的任务周数。
3. 计算北京时间当天所处周数。
4. 计算允许自动生成的最大周数：北京时间当天所处周数 + 1。
5. 若当前系统内最新任务周数已经等于或大于“北京时间当天所处周数 + 1”，则停止新增任务。
6. 若当前系统内最新任务周数小于“北京时间当天所处周数 + 1”，则新增下一周任务。

用户指定提示文案：

`本周任务已存在，若需新增，请手动处理`

该提示显示在自动任务模块下方。

### 5.7 周数示例

示例一：

- 当前系统已有第 19 周任务。
- 自动任务到达触发时间。
- 北京时间当天所处周数为第 19 周。
- 允许自动生成的最大周数为第 20 周。
- 系统自动新增第 20 周任务。

示例二：

- 当前系统已有第 20 周任务。
- 自动任务到达触发时间。
- 北京时间当天所处周数为第 19 周。
- 允许自动生成的最大周数为第 20 周。
- 系统不新增任务。
- 自动任务模块显示：`本周任务已存在，若需新增，请手动处理`。

### 5.8 跨年周数规则

自动任务必须正确处理跨年周数。

要求：

- 周数判断不能只比较 `week_number` 数字。
- 必须同时比较周所属年份与周数。
- 第 52 周或第 53 周之后，下一周可能进入下一年度第 1 周。
- 任务开始日期和结束日期必须与生成周的北京时间自然周一致。

### 5.9 新增任务内容

自动生成的任务必须复用现有任务创建口径。

新增任务字段要求：

- `time_dimension`：周维度。
- `year`：生成周所属年份。
- `week_number`：生成周周数。
- `start_date`：生成周周一。
- `end_date`：生成周周日。
- `status`：启用状态。
- `is_preferred`：成为当前首选任务。

任务标题沿用现有任务标题命名习惯。若现有手动创建任务标题格式为“第 N 周”或包含日期范围，则自动生成任务应保持一致风格。

### 5.10 首选任务与旧任务处理

自动新增任务时，必须保持当前 v3.0.2 已有首选任务规则：

- 新任务成为首选任务。
- 旧首选任务不再是首选任务。
- 不删除旧任务。
- 不删除旧任务下的填报数据。
- 不影响历史任务查看、报表和统计。

### 5.11 重复触发防护

系统必须避免同一规则在同一计划触发时间重复创建任务。

要求：

- 每次触发保存执行记录。
- 同一规则、同一计划时间只允许生成一次任务。
- 服务重启后不得因为调度恢复而重复补建同一时间点任务。
- 多条规则在同一秒触发时，也不得生成重复周任务。

### 5.12 自动任务状态显示

自动任务模块下方显示最近一次执行状态。

状态至少包括：

- 最近触发时间。
- 执行结果：成功、新增失败、无需新增、通知失败。
- 关联创建任务。
- 提示文案。

当无需新增时，必须显示：

`本周任务已存在，若需新增，请手动处理`

该提示持续显示，直到下一次配置时间再次触发并重新校验。

### 5.13 通知开关

每条自动任务规则后方有通知开关。

开关默认打开。

开关打开时：

- 显示钉钉 webhook 机器人配置输入框。
- 显示发送给钉钉 webhook 机器人的内容输入框。
- 显示确定编辑按钮。

开关关闭时：

- 不显示 webhook 输入框。
- 不显示消息内容输入框。
- 不显示确定编辑按钮。
- 自动任务仍可按时生成任务，但不发送钉钉通知。

### 5.14 钉钉 webhook 配置

webhook 配置输入框用于填写钉钉群机器人 webhook 地址或 webhook 配置信息。

要求：

- 支持粘贴完整 webhook 地址。
- 保存前进行基础格式校验。
- 不在页面明文展示额外派生密钥。
- 不将 webhook 地址写入前端静态文件。

### 5.15 钉钉消息内容

消息内容输入框用于填写发送给钉钉群机器人的固定文本内容。

本版本不新增模板占位符能力。  
若后续需要 `{taskTitle}`、`{weekNumber}`、`{startDate}` 等变量替换，需要另提需求。

### 5.16 保存交互

点击确定编辑按钮后：

- 保存成功：提示“编辑成功”。
- 保存失败：提示“编辑失败”。

保存失败时：

- 不覆盖旧配置。
- 显示失败原因。
- 不影响已存在的自动任务运行记录。

### 5.17 通知触发规则

仅在自动任务成功新增任务后，才发送钉钉 webhook 通知。

以下情况不发送通知：

- 未到配置触发时间。
- 触发后判断无需新增任务。
- 新增任务失败。
- 通知开关关闭。
- webhook 地址为空或校验失败。

发送失败时：

- 不回滚已创建任务。
- 记录通知失败状态。
- 页面显示通知失败结果。

### 5.21 2026-05-18 周任务生成口径修正

自动生成任务必须严格按“系统内已有最大周数 + 1”生成下一周任务，并受“当前日历周 + 1”封顶约束。

规则如下：

- 系统每次自动生成任务前，先读取系统内已有周维度任务。
- 以已有任务的 `year + week_number` 判断当前系统最大任务周。
- 下一次自动生成候选周 = 当前系统最大任务周 + 1。
- 若系统当前最大任务周是第 15 周，则下一次触发新增第 16 周。
- 若第 16 周被删除，系统最大任务周回到第 15 周，再次触发仍新增第 16 周。
- 若连续触发，生成顺序应为第 16 周、第 17 周、第 18 周、第 19 周、第 20 周，依次递增。
- 允许生成的最大周 = 北京时间当前日期所在自然周 + 1。
- 以 2026-05-18 为例，当天是 2026 年第 21 周，因此最高只允许生成到 2026 年第 22 周。
- 若候选周大于“当前日期所在自然周 + 1”，则不新增任务。
- 不新增任务时，继续显示现有提示：`本周任务已存在，若需新增，请手动处理`。

任务存在校验：

- 每次新增前必须单独校验候选周是否已经存在。
- 校验维度至少包含 `year + week_number`。
- 同时要校验候选周对应的周一到周日日期区间是否已存在周维度任务。
- 若对应周数或对应日期区间已存在，则继续寻找下一个候选周。
- 若寻找后的候选周超过“当前日期所在自然周 + 1”，则不新增任务，并显示：`本周任务已存在，若需新增，请手动处理`。

周日期口径：

- 系统中所有“周”相关计算统一为周一到周日。
- 周数按日历日期对应的自然周计算。
- 周任务标题、开始日期、结束日期、快捷周期、自动生成任务、任务筛选、统计归属、填报历史归属等涉及“周”的功能，必须统一使用周一到周日。
- 不允许再出现周日到周六的周周期。
- 本次优化不得改变历史已有任务数据本身，只修正后续计算、展示、生成和筛选口径。

## 6. REQ-32 需求工时统计全量数据备份下载

### 6.1 用户故事

作为管理员，我希望在设置页面一键下载系统内所有周期的需求工时统计数据，方便归档、审计和离线查看。

### 6.2 模块名称

设置页面新增模块：

`数据备份下载`

### 6.3 下载范围

下载范围限定为“需求工时统计”范围内的全部周期数据。

包含：

- 所有任务收集周期。
- 每个周期下的需求工时统计内容。
- 每条统计记录中的需求名称、版本、产品经理、前端、后端、测试、工时、备注等信息。
- 每个周期的小计。
- 全部周期的汇总信息。

不包含：

- 数据库物理备份。
- MySQL dump。
- 用户密码。
- 服务器文件。
- 前端源码或后端源码。
- 权限链接 token 明文清单。

### 6.4 文件格式

支持单文件下载。

管理员可选择：

- Excel 格式：`.xlsx`
- Markdown 格式：`.md`

默认格式建议为 Excel。

### 6.5 Excel 文件要求

Excel 下载为单个 `.xlsx` 文件。

建议结构：

- 第一个 Sheet：总览。
- 后续每个 Sheet：一个周期。

总览 Sheet 包含：

- 周期数量。
- 有统计数据的周期数量。
- 总工时。
- 按周期汇总的总工时。

周期 Sheet 包含：

- 周期标题。
- 年份。
- 周数。
- 开始日期。
- 结束日期。
- 需求名称。
- 版本。
- 产品经理。
- 前端人员与工时。
- 后端人员与工时。
- 测试人员与工时。
- 合计工时。
- 备注。

### 6.6 Markdown 文件要求

Markdown 下载为单个 `.md` 文件。

建议结构：

- 文档标题。
- 导出时间。
- 总览。
- 按周期分节。
- 每个周期内使用 Markdown 表格展示需求工时统计。

### 6.7 空周期处理

若某个任务收集周期暂无需求工时统计数据，也必须在文件中体现。

显示文案：

`暂无需求工时统计数据`

### 6.8 排序规则

周期排序应与系统任务列表的默认认知保持一致。

建议排序：

- 年份倒序。
- 开始日期倒序。
- 周数倒序。

### 6.9 下载接口建议

新增接口：

- `GET /api/settings/report-backup?format=xlsx`
- `GET /api/settings/report-backup?format=md`

接口要求：

- 返回单个文件下载。
- 文件名包含导出时间。
- 文件名示例：`需求工时统计全量备份_20260515_153000.xlsx`
- 下载动作需要权限 `btn:settings:backup_download`。

### 6.10 与现有导出功能关系

本功能不替代现有需求工时统计页面的单周期导出。

保留现有能力：

- 在需求工时统计页面导出当前选中周期。
- 导出后上传归档到现有 Excel 文件记录。

新增能力：

- 在设置页面下载全部周期的需求工时统计数据。

## 9. 非功能需求

### 9.1 安全

- webhook 地址不得写入前端静态代码。
- webhook 地址不得出现在浏览器控制台日志中。
- 下载全量报表需要权限控制。
- 设置页面需要权限控制。
- 不暴露数据库密码、服务器路径、token 明文清单。

### 9.2 稳定性

- 自动任务调度异常不得导致后端服务退出。
- 钉钉 webhook 调用失败不得回滚已创建任务。
- 下载全量数据失败不得影响现有报表页面。
- 新增设置页面失败不得影响任务、报表、统计、填报页面访问。

### 9.3 兼容性

- 不更换前端框架。
- 不更换后端框架。
- 不更换数据库。
- 不调整既有 API 的响应格式，除非本需求明确要求。
- 新增数据表不得破坏旧表结构。

### 9.4 可维护性

- 自动任务生成逻辑应抽取为后端服务函数，避免复制 `POST /api/tasks` 中的关键规则后产生行为差异。
- 周数计算应集中封装，避免多个位置重复实现跨年逻辑。
- 年份下拉建议集中封装工具函数，避免多个页面规则不一致。

## 11. 验收标准

### 11.1 设置入口验收

- 管理员身份下，顶部右侧“管理员”和“权限控制”之间出现齿轮设置入口。
- 点击齿轮进入 `/settings`。
- 设置页面包含“自动执行任务并通知”和“数据备份下载”两个模块。
- 无权限访问 `/settings` 时进入无权限页面。

### 11.2 自动任务验收

- 可以新增一条按年内每月日期执行的规则。
- 可以新增一条按每周星期执行的规则。
- 可以选择多个日期或多个星期。
- 可以精确配置到时分秒。
- 到达配置时间时，系统按北京时间判断是否需要新增下一周任务。
- 已有第 19 周任务且允许生成第 20 周时，系统自动新增第 20 周任务。
- 已有任务周数等于北京时间当前周数 + 1 时，不新增任务。
- 不新增时显示：`本周任务已存在，若需新增，请手动处理`。
- 自动新增任务后，新任务成为首选任务。
- 自动新增任务不删除旧任务和旧数据。
- 同一触发时间不会重复创建任务。

### 11.3 钉钉 webhook 验收

- 通知开关默认打开。
- 开关打开时显示 webhook 输入框、消息内容输入框、确定编辑按钮。
- 开关关闭时隐藏 webhook 输入框、消息内容输入框、确定编辑按钮。
- 保存成功提示“编辑成功”。
- 保存失败提示“编辑失败”。
- 自动新增任务成功且通知开关打开时，向配置 webhook 发送预设内容。
- 判断无需新增任务时，不发送 webhook 通知。
- webhook 发送失败不回滚已创建任务。

### 11.4 数据备份下载验收

- 设置页面可以选择 Excel 或 Markdown 格式。
- 点击下载后得到单个文件。
- 文件包含所有任务周期。
- 每个周期按需求工时统计口径列出数据。
- 空周期显示“暂无需求工时统计数据”。
- 现有需求工时统计页面单周期导出能力不受影响。

### 11.5 年份下拉验收

- 当前实际年份为 2026 时，任务收集列表年份下拉可选择 2027。
- 当前实际年份为 2026 时，周期统计页面年份下拉可选择 2027。
- 当前实际年份为 2026 时，填报历史年份下拉可选择 2027。
- 默认仍选中 2026。
- 选择 2027 时无数据页面按现有空状态显示。

### 11.6 页脚验收

- 页面内容较少时，底部信息显示在浏览器底部。
- 页面内容较多时，底部信息在内容之后，不遮挡内容。
- 后台页面与填报页面均生效。
- 文案显示为：`DevTracker v3.0.2© 2026   2698-jfzhu8023`。

### 11.7 回归验收

必须验证以下历史功能不受影响：

- 手动创建任务。
- 任务列表筛选与查看。
- 任务详情填报记录查看。
- 填报链接提交。
- 需求工时统计查看与导出。
- 周期统计查看与导出。
- 人员管理。
- 权限控制。
