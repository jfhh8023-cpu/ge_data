## 3. REQ-42 新增规则前选择自动执行任务类型

### 3.1 触发入口

点击设置页“自动执行任务并通知”模块右上角 `新增规则` 按钮时，不再直接新增默认规则，而是先弹出任务类型选择弹窗。

### 3.2 弹窗要求

弹窗标题：

`请选择一个自动执行任务类型`

弹窗可选项：

1. `自动任务创建并通知`
2. `自动值班通知`

交互要求：

- 弹窗内以两个可点击选项展示。
- 点击选项后关闭弹窗，并新增对应类型任务。
- 取消或关闭弹窗时不新增任何任务。

## 4. REQ-43 自动任务创建并通知标题标识

选择 `自动任务创建并通知` 后：

- 新增逻辑沿用当前自动执行任务并通知任务。
- 当前已有执行模式、周/月调度、通知子模块、接收人、测试发送 webhook 等逻辑和内容不变。
- 规则行序号后新增标题：`自动任务创建并通知`。
- 该标题仅作为任务类型标识，不影响已有规则执行。

建议后端类型值：

`task_create_notify`

旧数据兼容：

- 旧规则未保存类型时，默认视为 `task_create_notify`。

## 5. REQ-44 自动值班通知任务

### 5.1 任务定位

`自动值班通知` 是新的通知类自动任务。

它不负责创建下一周任务收集任务，只负责按配置的星期或日期，在指定时间向钉钉 webhook 发送值班通知。

建议后端类型值：

`duty_notify`

### 5.2 新增后的任务标题

选择 `自动值班通知` 后：

- 序号继续按规则列表自增。
- 序号后展示标题：`自动值班通知`。
- 与 `自动任务创建并通知` 在视觉上明显区分。

### 5.3 第一行公共配置

自动值班通知第一行包含：

- 序号。
- 类型标题：`自动值班通知`。
- 调度类型下拉：
  - `每周星期`
  - `年内每月日期`
- 启用开关。
- 保存通知。
- 测试发送webhook。
- 删除。

说明：

- 自动值班通知不展示 v3.1.2 的“执行规则并通知 / 仅执行规则 / 仅通知”执行模式，因为它天然属于“仅通知”类型。
- 自动值班通知不展示“测试执行”，避免误导为会生成任务。

### 5.4 每周星期模式

选择 `每周星期` 时，任务主体展示周一至周日的配置行。

每一天可配置：

- 是否启用当天。
- 单个或多个团队人员。
- 单个或多个触发时间。
- 通知内容输入框，支持换行。

示例：

| 星期 | 值班人员 | 通知时间 | 通知内容 |
| --- | --- | --- | --- |
| 周一 | 张三、李四 | 09:00:00、18:00:00 | 今日值班提醒... |
| 周二 | 王五 | 09:30:00 | 请关注线上告警... |

执行规则：

- 到达某天某个配置时间时，若该天启用且存在通知内容，则发送 webhook。
- 若配置了值班人员，通知内容中可展示值班人员姓名，并按现有钉钉接收人能力 @ 对应人员。
- 同一天多个时间应分别触发。
- 同一天多个值班人员使用同一条通知内容。

### 5.5 年内每月日期模式

选择 `年内每月日期` 时，任务主体展示日期配置。

配置范围：

- 每月 1 号至 31 号。
- 触发时按实际日历日期判断；不存在的日期不触发，例如 2 月 30 日不触发。

每个日期可配置：

- 是否启用该日期。
- 单个或多个团队人员。
- 单个或多个触发时间。
- 通知内容输入框，支持换行。

执行规则：

- 到达每月指定日期和时间时发送 webhook。
- 按自然日历判断日期是否存在。
- 不存在日期自动跳过，不报错、不补发。

### 5.6 预览、编辑、保存、删除、测试

自动值班通知单任务界面必须支持：

- 配置信息可见。
- 可预览当前选中的星期/日期值班通知。
- 可编辑。
- 可保存通知。
- 可删除任务。
- 可测试发送 webhook。

测试发送 webhook：

- 不等待真实调度时间。
- 使用当前预览或当前第一条有效配置发送测试通知。
- 不生成任务收集任务。

删除：

- 必须二次确认。

## 6. REQ-45 删除规则二次确认

删除按钮点击时必须二次确认。

覆盖范围：

- 自动任务创建并通知。
- 自动值班通知。

确认文案建议：

- 标题：`删除规则`
- 内容：`确认删除「任务类型/规则名称」？删除后不可恢复。`
- 确认按钮：`删除`
- 取消按钮：`取消`

说明：

- 当前自动任务创建并通知已有删除确认；后续开发需确保自动值班通知也使用同一确认机制。
- 若新增规则尚未保存到后端，删除本地草稿时也建议二次确认，避免误删刚配置的大量值班数据。

## 7. REQ-46 自动值班通知差异化布局

### 7.1 设计目标

自动值班通知需要区别于“自动任务创建并通知”的单行规则 + 子模块布局，同时必须控制高度。

推荐采用“紧凑周期概览”布局：

- 顶部保持紧凑规则头。
- 单条自动值班通知任务整体高度不得超过“自动任务创建并通知”单条任务高度的 1.5 倍。
- 不采用按天纵向展开的大表格，避免 7 天或 31 天数据直接撑高页面。
- 人员、时间、通知内容在列表内展示摘要和当前选中项预览，完整编辑进入详情弹窗或抽屉。

### 7.2 每周星期布局建议

结构：

```text
序号 自动值班通知 | 每周星期 | 值班计划概览 | 保存通知 | 测试发送webhook | 删除 | 启用
└─ 每周值班概览：周一 周二 周三 周四 周五 周六 周日
   每个星期格展示：人员摘要、时间摘要、配置状态
   预览：当前选中星期 09:00:00 @张三 @李四 通知内容前若干字 | 编辑详情
```

布局特点：

- 7 天全部在同一横向概览区域内展示。
- 每个星期格只展示摘要，避免单独展开多行。
- 内容可换行编辑，但列表内只展示预览文本。

### 7.3 年内每月日期布局建议

结构：

```text
序号 自动值班通知 | 年内每月日期 | 值班计划概览 | 保存通知 | 测试发送webhook | 删除 | 启用
└─ 每月日期概览：1日 2日 3日 ... 31日
   已配置日期高亮，未配置日期弱化
   预览：当前选中日期 09:30:00 @张三 @李四 通知内容前若干字 | 编辑详情
```

布局特点：

- 1-31 日必须在同一紧凑区域内完整展示，优先控制为两行。
- 已配置日期高亮；未配置日期弱化；自然月不存在的日期执行时自动跳过。
- 可通过 `+ 日期配置` 或点击日期格进入编辑详情。

### 7.4 开始/结束提醒与批量内容编辑

自动值班通知配置详情弹窗内必须支持：

- 值班开始提醒时间和开始提醒内容。
- 值班结束提醒时间和结束提醒内容。
- 发送策略二选一：`只发送开始提醒`、`开始和结束都发送`。
- 选择 `只发送开始提醒` 时，仅在开始时间触发开始提醒内容。
- 选择 `开始和结束都发送` 时，在开始时间发送开始提醒内容，在结束时间发送结束提醒内容。
- 团队人员勾选项采用紧凑尺寸展示，避免弹窗占用过高。

月模式必须支持批量编辑所有日期通知内容：

- 可选择覆盖 `所有日期`、`仅已配置日期` 或 `自定义日期`。
- 批量编辑只覆盖开始/结束提醒文案和发送策略，不改变每个日期原有人员与时间配置。
- 保存后应用于目标日期范围。

### 7.5 webhook 发送正文与@人规则

- `今日值班开始提醒：` 和 `今日值班结束提醒：` 仅作为配置弹窗内输入框标题展示。
- webhook 实际发送正文只使用输入框内填写的内容，不把上述标题拼入正文。
- 自动值班通知发送时，当前值班配置中勾选的团队人员必须被 @。
- @ 人依赖团队人员手机号；手机号为空的人员不可勾选为值班通知接收人。
- 钉钉消息页面样式参考“自动任务创建并通知”的 markdown 卡片样式，保持统一标题、正文、@人展示结构。

### 7.6 推荐 Demo 方案

本轮提供静态 Demo：

`docs/@plan/demo_v3.1.3_auto_duty_task.html`

Demo 内容包含：

- 自动值班通知每周星期布局示例。
- 自动值班通知年内每月日期布局示例。
- 删除二次确认交互示意。

## 8. REQ-50 自动值班通知补充优化

### 8.1 每周/每月批量配置

- 每周值班和每月值班都必须支持批量填写通知内容。
- 每周值班和每月值班都必须支持引用某一天/某星期的完整配置。
- 批量填写仅覆盖发送策略、开始提醒内容、结束提醒内容，不改变已配置的值班人员和时间。
- 引用配置会复制来源项的值班人员、开始时间、结束时间、发送策略、开始提醒内容、结束提醒内容与启用状态。
- 配置详情弹窗内也必须可以选择引用同一规则下某一天/某星期的配置内容。

### 8.2 开始/结束提醒显示规则

- 选择 `只发送开始提醒` 时，仅展示值班开始时间和今日值班开始提醒输入框。
- 选择 `开始和结束都发送` 时，展示值班开始时间、值班结束时间、今日值班开始提醒、今日值班结束提醒。
- 发送 webhook 时，只有选择 `开始和结束都发送` 且到达结束时间，才发送结束提醒。

### 8.3 人员名称与 @ 人

- 配置弹窗、概览、预览优先展示团队人员姓名。
- 若历史配置中保存的是手机号而不是人员 id，需要尽量按团队人员手机号反查姓名。
- 钉钉 webhook 发送时仍按团队人员手机号 @ 人；不在群内导致无法点击属于钉钉群成员限制，不影响系统展示姓名。

### 8.4 任务互不干涉

- 每条自动值班通知规则按自己的配置独立计算触发事件。
- 同一条规则的开始提醒和结束提醒应作为不同事件记录，避免同一时间或相邻时间触发时互相抢占日志。
- 某条规则 webhook 发送失败，不得阻断其他规则继续执行。

### 8.5 界面细节

- `编辑详情` 按钮需要比普通小按钮更明显，便于点击。
- `钉钉 webhook 通知` 后面的内容预览应尽量显示完整内容，不再只展示 20 字截断。
- 自动任务创建并通知、自动值班通知两类任务下的 webhook 名称和地址输入区域长度保持一致。
- 自动值班通知钉钉 markdown 卡片标题改为：`今日语音业务线值班通知：`。
- 自动值班通知第一行的 `每周星期 / 年内每月日期` 下拉框采用短宽度，只需略宽于最长选项文字。
- 自动任务创建并通知的周下拉框、时间框、执行模式下拉框压缩宽度，右侧保存、测试、删除、启用开关靠右。
- 自动任务创建并通知的 `钉钉 webhook 通知` 开关移动到子模块操作区，放在 `测试发送webhook` 后面；通知关闭时也必须保留开关以便重新打开。

## 9. 后续开发建议

### 9.1 后端模型

在现有 `auto_task_rules` 上兼容扩展：

- `task_type VARCHAR(40) NOT NULL DEFAULT 'task_create_notify'`
- `duty_config TEXT NULL`

在现有 `auto_task_run_logs` 上兼容扩展：

- `event_type VARCHAR(30) NOT NULL DEFAULT 'auto_task'`
- 唯一约束建议调整为 `(rule_id, scheduled_at, event_type)`，用于区分同一规则下不同触发事件。

`duty_config` 建议 JSON 结构：

```json
{
  "mode": "weekly",
  "weekly": {
    "1": {
      "enabled": true,
      "staff_ids": ["staff-id-1"],
      "times": ["09:00:00"],
      "message": "今日值班提醒..."
    }
  },
  "monthly": {
    "1": {
      "enabled": true,
      "staff_ids": ["staff-id-1"],
      "times": ["09:00:00"],
      "message": "月初值班提醒..."
    }
  }
}
```

### 9.2 调度逻辑

- `task_create_notify`：沿用当前自动创建任务逻辑。
- `duty_notify`：只执行值班通知调度。
- 两类任务共用启用开关、webhook 配置、接收人解析能力。

### 9.3 兼容策略

- 旧规则默认 `task_type = task_create_notify`。
- 旧规则不需要 `duty_config`。
- 新增字段通过缺列补列方式迁移，不重建表。

## 10. 验收标准

### 10.1 Demo 验收

- 点击新增规则先出现“请选择一个自动执行任务类型”弹窗。
- 弹窗有 `自动任务创建并通知` 和 `自动值班通知` 两个选项。
- 自动任务创建并通知示例与现有布局保持一致，并在序号后显示标题。
- 自动值班通知示例采用紧凑周期概览布局，明显区别于现有规则布局。
- 自动值班通知支持每周星期和年内每月日期两种布局展示。
- 自动值班通知整体高度不超过自动任务创建并通知高度的 1.5 倍。
- 点击星期格、日期格、编辑详情、日期配置、批量复制时，必须弹出对应配置界面。
- 配置弹窗支持开始提醒内容、结束提醒内容，以及“只发送开始提醒 / 开始和结束都发送”策略。
- 月模式支持批量编辑所有日期通知内容。
- 团队人员勾选框采用紧凑尺寸，不得明显撑高弹窗。
- webhook 实际发送正文只使用输入框内容；开始/结束提醒标题不进入正文；勾选人员必须被 @。
- 删除按钮有二次确认示意。

### 10.2 后续开发验收

- 每周值班可以批量填写通知内容，可以引用某个星期的完整配置。
- 每月值班可以批量填写通知内容，可以引用某个日期的完整配置。
- 详情弹窗内可以引用同规则下某一天/某星期配置。
- 只发送开始提醒时，不展示结束时间和结束提醒输入框。
- 开始和结束都发送时，才展示结束时间和结束提醒输入框。
- 团队人员有姓名时，配置弹窗、概览、预览优先显示姓名；历史手机号配置尽量反查姓名。
- 自动值班通知发送到钉钉的卡片标题为 `今日语音业务线值班通知：`。
- 多条自动值班通知规则触发时，某一条失败不影响其他规则。
