# DevTracker v1.4.2 — 多页面 Bug 修复 + 功能增强

> 日期：2026-04-17
> 版本：v1.4.2

## 变更概要

修复 7 类问题，涵盖周期统计表格错位、卡片排序、合计格式、填写页留白/按钮、需求工时统计周期逻辑和编辑提示。

## 变更清单

### 1. StatsPage.vue — 周期统计页

| 需求 | 改动 |
| ---- | ---- |
| 明细表列错位修复 | `pmSpanMethod` 总计行 colspan 从 4 改为 5，覆盖版本号→工时全部 5 列 |
| 概要卡片排序 | 调整为：总工时 → 前端 → 后端 → 测试 → 记录数 → 人员 → 任务数 |
| 合计行文本 | 去掉"一共"，改为 `合计     xxx.xx小时` |
| 个人聚焦-单独查看修复 | 所有列添加 `prop` 绑定、表格加 `table-layout="fixed"` |

### 2. FillPage.vue — 填写工时页

| 需求 | 改动 |
| ---- | ---- |
| 留白再减半 | `.fill-page-root` padding `30px 20px` → `30px 10px` |
| 按钮加大 | `min-width:140px; height:44px; font-size:16px; font-weight:600` |
| 收集状态标签增强 | 字号 14px、字重 600；active 深绿色 `#006B1A` on `#D4FFDA`；closed 红色 `#CB2634` on `#FFE8E8` |

### 3. main.css — 全局样式

- `.dt-fill-layout` padding `40px 50px` → `40px 25px`

### 4. ReportPage.vue — 需求工时统计页

| 需求 | 改动 |
| ---- | ---- |
| 编辑模式"不可编辑"提示 | 非 `manual_merged` 行操作列显示 `不可编辑` 标签替代 `-` |
| 本周/上周逻辑重写 | 新增 `taskContainsToday()`、`taskIsLastWeek()` 工具函数 |
| 默认选中修正 | 优先上周 → 若有本周任务则选本周 → 兜底最新 |
| 标题前缀修正 | 基于任务日期范围判断"本周/上周"，不再依赖列表索引 |

### 5. App.vue — 版本号

- `v1.4.1` → `v1.4.2`
