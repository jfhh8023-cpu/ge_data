# DevTracker v1.4.4 — 实时报表 + 个人聚焦优化 + 图表修复

> 日期：2026-04-17
> 版本：v1.4.4

## 变更概要

修复 3 类问题：需求工时统计页不实时展示新提交数据、周期统计个人聚焦任务列表展示不全且排序不正确、部门全观柱状图偶现不渲染。

---

## 变更清单

### 1. 需求工时统计页实时更新（Issue #1）

**问题**：用户通过填写页提交工时后，需求工时统计页（ReportPage）不会实时反映新数据，需要手动触发匹配。

**根因**：填写提交只创建 `WorkRecord`，但 ReportPage 读取的是 `MatchGroup` 表，两者之间没有自动同步。

**修复文件**：`backend/src/routes/fill.js`

| 改动 | 说明 |
| ---- | ---- |
| 自动触发匹配 | 提交成功后自动执行 `matchRecords` + 重建 `MatchGroup`，使 ReportPage 刷新即可看到最新数据 |
| 异常隔离 | 匹配失败不影响提交响应，仅记录日志 |

### 2. 周期统计-个人聚焦-单独查看优化（Issue #2）

**问题**：个人聚焦只展示有记录的任务周期、无排序、默认展开第一个任务。

**修复文件**：
- `backend/src/routes/stats.js` — 个人统计端点
- `frontend/src/views/StatsPage.vue` — 个人聚焦视图

| 改动 | 说明 |
| ---- | ---- |
| 展示所有任务周期 | 后端返回所有任务（含无记录的），而非仅有记录的 |
| 按周期倒序 | 后端按 `start_date` 降序排列，最新周期在最上方 |
| 默认展开上一周 | 前端根据当前日期计算上周一，匹配 `start_date` 自动展开 |
| 无记录周期样式 | 无记录行半透明 + 标签显示"暂无记录"，展开显示空状态提示 |

### 3. 部门全观柱状图首次加载修复（v1.4.3 遗留）

**问题**：柱状图偶现不显示，需切换 Tab 才出现。

**根因**：`drawChart()` 在 `pageLoading` 从 true→false 之前执行，此时 canvas 还在 `v-if` 隐藏分支中。

**修复文件**：`frontend/src/views/StatsPage.vue`

| 改动 | 说明 |
| ---- | ---- |
| 延迟绘制 | `pageLoading=false` 后 `nextTick` + `setTimeout(drawChart, 50)` |
| Tab 切换重绘 | `watch(activeTab)` 切回部门全观时自动 `nextTick` + `drawChart()` |
