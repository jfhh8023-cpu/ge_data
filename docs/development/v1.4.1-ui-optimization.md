# DevTracker v1.4.1 — UI 优化与性能提升

> 日期：2026-04-17

## 变更概要

基于用户反馈的 6 项优化需求，涵盖填写页 UI 调整、周期统计表改造、个人聚焦修复及全站性能提升。

## 变更清单

### 1. FillPage.vue — 填写页 UI 优化

| 需求 | 改动 |
|------|------|
| 收集状态标签 | 标题区右上角新增 `正在收集中` / `收集已停止` 状态标签，绿色/灰色圆点，active 状态带呼吸动画 |
| 按钮加大 | 暂存草稿/提交按钮从 `size="small"` 改为 `size="default"`，增加 `min-width:120px`、`font-size:15px`、`height:40px` |
| 留白减半 | `.fill-page-root` padding 从 `30px 40px` 改为 `30px 20px` |

### 2. main.css — 全局样式

- `.dt-fill-layout` padding 从 `40px 100px` 改为 `40px 50px`

### 3. StatsPage.vue — 周期统计优化

| 需求 | 改动 |
|------|------|
| 移除总计列 | 删除独立的"总计/小时"列 |
| 合计行改造 | 总计行从版本号列开始合并 4 列，显示 `合计   一共xxx.xx小时` |
| 单独查看修复 | 产品经理列改为 `min-width="130"`，工时列宽度改为 `100`，标题改为"工时(小时)" |
| 交替背景色 | 一起查看模式三列（后端/前端/测试）人员块奇偶行交替背景色 |

### 4. vite.config.js — 性能优化

- 新增 `manualChunks` 函数，将 Vue 生态和 Element Plus 分离为独立 chunk
- **注意**：Vite 8 + Rolldown 要求 `manualChunks` 为函数形式（不支持对象语法）

构建产物分离效果：
- `vendor-vue` → 30.74 kB (gzip: 12.06 kB)
- `vendor-ui` → 948.72 kB (gzip: 306.11 kB)

### 5. App.vue — 版本号

- `v1.4.0` → `v1.4.1`
