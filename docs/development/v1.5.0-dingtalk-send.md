## 功能概述

在「任务详情 → 链接管理」Tab 中，为全员生成链接按钮右侧新增钉钉登录状态指示器和二维码扫码入口，管理员可一键将专属填写链接通过钉钉私聊发送给对应员工。

## API 接口

| 接口 | 方法 | 说明 |
|------|------|------|
| `/api/dingtalk/status` | GET | 查询登录状态和会话年龄 |
| `/api/dingtalk/login/qrcode` | POST | headless 截取二维码图片（base64）并启动后台监听 |
| `/api/dingtalk/login/poll` | GET | 轮询扫码是否完成 |
| `/api/dingtalk/send` | POST | 发送私聊消息 |

## 前端 UI 交互说明

### 状态指示器（「为全员生成链接」按钮右侧）

- **绿色「钉钉已登录 · X天前」**：会话有效，点击可重新扫码（换账号）
- **红色「钉钉未登录」**：无会话文件，点击展开二维码面板

### 二维码面板（▼ 扫码 / ▲ 收起 按钮）

1. 点击按钮 → 调 `POST /api/dingtalk/login/qrcode`
2. headless Chromium 打开钉钉登录页，截取二维码区域返回 base64
3. 前端渲染二维码图片，开始每 2 秒轮询 `/api/dingtalk/login/poll`
4. 用户用手机钉钉扫码 → 后端检测 URL 跳转 → 保存 session
5. 轮询返回 `done: true` → 状态变绿「已登录」→ 面板自动关闭

### 发送按钮

- 点击「发送」→ `POST /api/dingtalk/send { staffName, message }`
- 消息格式：
  ```
  【工时填写提醒】
  {任务标题}
  请点击链接填写：{专属URL}
  ```
- 发送中：按钮显示「发送中...」+ loading，其他行禁用
- 成功：`ElMessage.success('已发送给 xxx')`
- 会话过期：状态变红 + 提示重新扫码
- 未找到联系人：精确提示姓名

## 技术实现要点

### headless 截图方案（vs. 原 headed 弹窗方案）

原 DEV 文档设计为 `headless: false` 弹出独立浏览器窗口扫码。本次改为：

1. `headless: true` 后台运行
2. 截取二维码元素截图（优先截 `img` 元素，降级截视口区域）
3. 以 base64 返回给前端，嵌入页面内显示
4. 后台保持浏览器实例监听扫码跳转事件

**优势**：完全无感，不弹窗口，不抢占焦点，用户体验更佳。

### 串行发送队列

`DingtalkService._sendQueue` 是一个 Promise 链，确保多次点击发送时串行执行，不会并发启动多个 Chromium 实例。

## 测试指引

> 仅对「朱俊锋」账户进行测试，其他人员暂不点击发送。

### 验证步骤

1. 本地启动后端：`cd backend && node src/app.js`
2. 本地启动前端：`cd frontend && npx vite`
3. 打开任意周期任务详情 → 切换到「链接管理」Tab
4. 观察「为全员生成链接」右侧的钉钉状态指示器（应显示红色「未登录」）
5. 点击「▼ 扫码」按钮 → 等待二维码图片出现
6. 用手机钉钉扫码 → 等待状态自动变绿「已登录」
7. 找到朱俊锋所在行 → 点击「发送」
8. 验证朱俊锋的钉钉收到消息

### 常见问题排查

| 现象 | 可能原因 | 解决方案 |
|------|----------|----------|
| 二维码获取失败 | 钉钉网页版 DOM 结构变化 | 更新 `dingtalk-selectors.js` |
| 二维码是整页截图 | QR_CODE_IMG 选择器不匹配 | 用 codegen 重新定位选择器 |
| 发送失败「未找到联系人」 | 朱俊锋不在登录账号的钉钉通讯录 | 确认账号和联系人在同一组织 |
| 会话立即过期 | storageState 保存不完整 | 扫码后再等待 2-3 秒再保存 |
