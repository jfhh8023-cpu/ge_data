# DevTracker 项目架构理解文档 v3.2.0

> 归档时间：2026-06-10

## 项目概览

DevTracker 是一个**轻量级 SaaS 工时采集与多维统计系统**，当前版本 v3.2.0。  
生产环境部署在 `jfzhu8023.cloud/devtracker/`。

## 技术栈

- **前端**: Vue 3.5 + Vite 8 + Element Plus + Pinia 3 + vue-router 4 + axios + xlsx(SheetJS) + file-saver + sortablejs
- **后端**: Node.js + Express 5 + Sequelize 6 ORM + mysql2 + multer + xlsx + playwright + uuid
- **数据库**: MySQL/MariaDB，UTF-8MB4
- **部署**: Nginx + PM2 + Python 自动化脚本(`deploy/deploy.py`)
- **开发端口**: 前端 5173（Vite dev server, proxy `/api` → 3001），后端 3001

## 数据模型（13 个 Sequelize Model）

| 模型 | 说明 |
|:---|:---|
| CollectionTask | 收集任务（按周/季度等维度） |
| Staff | 人员（前端/后端/测试三角色） |
| FillLink | 填写链接（每人每任务一个 token） |
| WorkRecord | 原始工时记录（人+需求+版本号+工时） |
| MatchGroup | 智能匹配归并后的需求维度汇总 |
| AccessLink | 访问链接（权限控制入口） |
| LinkPermission | 链接权限明细（40 个资源×CRUD） |
| StaffFillLink | 系统级填写链接（v1.6.0） |
| ExcelFile | Excel 文件存档（导入导出） |
| ProductManager | 产品经理 |
| AutoTaskRule | 自动任务规则（v3.1.0） |
| AutoTaskRunLog | 自动任务执行日志 |
| AutoTaskMessage | 自动任务消息（钉钉通知等） |

### 核心关联

- Task 1:N FillLink / WorkRecord / MatchGroup
- Staff 1:N FillLink / WorkRecord
- AccessLink 1:N LinkPermission
- AutoTaskRule 1:N RunLog / Message

## 后端路由（10 个模块）

| 路由前缀 | 文件 | 职责 |
|:---|:---|:---|
| `/api/staff` | staff.js | 人员 CRUD |
| `/api/tasks` | tasks.js | 收集任务管理 |
| `/api/records` | records.js | 工时记录 + 批量导入 |
| `/api/report` | report.js | 需求工时统计 + 导入 |
| `/api/fill` | fill.js | 填写页接口 |
| `/api/stats` | stats.js | 周期统计 |
| `/api/permissions` | permissions.js | 权限管理 |
| `/api/excel` | excel.js | Excel 上传/模板下载 |
| `/api/settings` | settings.js | 系统设置(含自动任务规则) |
| `/api/pm` | pm.js | 产品经理管理 |

## 核心服务

- **MatchService** — 智能匹配引擎：Phase1 按版本号精确分桶，Phase2 无版本号用相似度模糊匹配
- **AutoTaskService** — 自动任务调度器：定时创建收集任务 + 钉钉通知（值班/进度维护）
- **TaskService** — 任务创建服务
- **ReportBackupService** — 报表备份服务

## 权限体系

- 两种访问模式：管理员(`?admin=1`) / 权限链接(`?token=xxx`)
- 40 个资源 × 4 操作，按 6 个模块分组
- FillPage 通过专属 token 访问，不受 AccessLink 权限限制
- 路由守卫 + 按钮守卫双层控制
