# 开发计划：柱状图标签纵向优化 + 名句搭配功能

> 创建时间：2026-06-10  
> 版本目标：v3.3.0  
> 本次优化不发布生产

---

## 需求概述

### 需求 A：StatsPage 柱状图标签纵向显示

**现状**：每根柱子上方标签为 `角色名 数值` 横向显示（如 `前端 120`），当产品经理较多时横向文字相互遮挡。

**目标**：改为纵向布局 — 上方为角色名文字，下方为数字，两者左对齐。避免标签横向溢出遮挡。

### 需求 B：名句搭配功能（值班通知增强）

**位置**：SettingsPage → 自动值班通知规则 → 每周值班概览卡片左侧，值班对象下方空白位置。

**功能清单**：

1. **开关**：开启名句搭配 / 关闭名句搭配
2. **随机选取**：开启后，从句子库中随机获取一个句子，插入到今日值班 开始/结束 提醒文案前方（先句子，换行后跟原始内容）
3. **开始和结束不同句**：每次值班开始提醒和结束提醒各取一个不同的句子
4. **去重机制**：近 N 次句子提取不得重复（默认 N=20），N 可在界面手动修改
5. **候选队列**：显示接下来近 10 次将被使用的句子候选；最近一次用框框着重体现
6. **操作按钮**：可点击"更换一批"重新随机生成候选队列，以及"使用下一句"跳过当前
7. **查看全部**：底部按钮打开大弹窗，按顺序显示所有句子（最近一次要被使用的着重体现）
8. **句子管理**：弹窗中可添加新句子、编辑、删除（支持单选/多选批量删除）
9. **默认句子**：系统内置约 300 条对联式句子（用户提供）

## 效果预览

### 预览 A：柱状图柱顶标签 — 改动前后对比

#### 改动前（现状，横向标签）

每根柱子上方为 `角色名 数值` 一行横排，PM 名字一多时相邻柱子的标签互相覆盖：

```
                                                                     
    前端 12  后端 8  测试 5  总计 25                                 
       ▌       ▌       ▌       ▌                                    
       █       █       █       █     前端 9  后端 11  测试 4  总计 24
       █       █       █       █        ▌       ▌       ▌       ▌  
       █       █       █       █        █       █       █       █  
       █       █       █       █        █       █       █       █  
    ─────────────────────────────────────────────────────────────────
                    杨瑞                              钟冠           
```

问题：当 PM 数量多、柱子变窄时，`前端 12  后端 8` 这种横排标签会越过柱子宽度，挤到相邻柱子上方，互相遮挡读不清。

#### 改动后（纵向标签，左对齐）

角色名在上、数值在下，两行均与柱子左边缘对齐；标签竖向占空间，水平方向只占一行字符宽度，不再外溢：

```
                                                                     
    前端    后端    测试    总计                                     
    12      8       5       25                                       
       ▌       ▌       ▌       ▌                                    
       █       █       █       █     前端    后端    测试    总计   
       █       █       █       █     9       11      4       24     
       █       █       █       █        ▌       ▌       ▌       ▌  
       █       █       █       █        █       █       █       █  
       █       █       █       █        █       █       █       █  
    ─────────────────────────────────────────────────────────────────
                    杨瑞                              钟冠           
```

**关键变化**：

- 标签 textAlign 由 `'center'` 改为 `'left'`，锚点改为柱子的左边缘 x
- 单行 `前端 12` 拆为两行：第一行 `前端`，第二行 `12`
- 行间距 ~12px，padding.top 由 45 增至 ~65 以预留两行高度
- 横向最大宽度 = 单个角色名 / 单个数字字符宽度，远小于原来的 `前端 12` 拼接宽度
- 即使 PM 数量很多、柱子很窄，标签也只是稍稍向左探出柱子半个字宽，不会跨柱遮挡

### 预览 B：钉钉值班通知 — 改动前后对比

#### 改动前（现状）

仅发送配置中填写的「今日值班开始/结束提醒」文本：

> **今日语音业务线值班通知：**
>
> 请关注线上告警和待处理反馈。

#### 改动后（开启名句搭配）

**值班开始通知**（取候选队列第 1 条句子）：

> **今日语音业务线值班通知：**
>
> 伴月披星冲霄汉，迎风冒雨不知寒。
>
> 请关注线上告警和待处理反馈。

**同一天的值班结束通知**（取候选队列第 2 条，保证与开始句不同）：

> **今日语音业务线值班通知：**
>
> 案前灯火燃深夜，笔底波澜写汗青。
>
> 请同步今日值班处理结果。

**关键规则**：

- 句子放在第一行，原 `start_message` / `end_message` 换行追加在下方
- 开始与结束取自候选队列的不同位置（[0] 和 [1]），同一天必然不同
- 已使用句子写入 `used_history`，候选队列从头部移除并从未使用池中补一条新句到末尾
- 近 N 次（默认 20）不重复：每次随机抽样时排除 `used_history` 最近 N 条

## 技术方案

### 需求 A：柱状图标签（纯前端 Canvas 改动）

**改动文件**：`frontend/src/views/StatsPage.vue`

**改动位置**：`drawChart()` 函数中绘制柱顶标签部分（约第 485-490 行）

**改动内容**：
- 原来：`ctx.fillText(\`${BAR_LABELS[bi]} ${val.toFixed(0)}\`, x + barWidth/2, y - 4)` 一行横排
- 改为：分两行绘制
  - 第一行：角色名（如"前端"），左对齐于柱子左边缘 x
  - 第二行：数值（如"120"），左对齐于柱子左边缘 x，位于角色名正下方
  - textAlign 改为 'left'
  - 两行间距约 12px
  - padding.top 需要适当增大（从 45 → ~65）以容纳两行标签

---

### 需求 B：名句搭配功能

#### Phase 1：后端 — 句子资料库与队列管理

**新建文件**：
- `backend/src/models/Quote.js` — 名句模型（id, content, sort_order, created_at）
- `backend/src/routes/quotes.js` — 名句 CRUD + 队列接口

**修改文件**：
- `backend/src/models/index.js` — 注册 Quote 模型
- `backend/src/app.js` — 注册 `/api/quotes` 路由

**新增数据库表 `quotes`**：
```sql
CREATE TABLE IF NOT EXISTS quotes (
  id CHAR(36) PRIMARY KEY,
  content TEXT NOT NULL COMMENT '句子内容',
  sort_order INT DEFAULT 0 COMMENT '排序',
  created_at DATETIME DEFAULT CURRENT_TIMESTAMP
) ENGINE=InnoDB COMMENT='名句库';
```

**新增数据库表 `quote_config`**：
```sql
CREATE TABLE IF NOT EXISTS quote_config (
  id CHAR(36) PRIMARY KEY,
  rule_id CHAR(36) NOT NULL COMMENT '关联的自动任务规则ID',
  enabled BOOLEAN DEFAULT FALSE COMMENT '是否开启名句搭配',
  no_repeat_count INT DEFAULT 20 COMMENT '近N次不重复',
  candidate_queue JSON COMMENT '候选队列 [quote_id, ...]',
  used_history JSON COMMENT '已使用历史 [quote_id, ...]（按时间倒序）',
  updated_at DATETIME DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
  UNIQUE KEY uniq_rule (rule_id)
) ENGINE=InnoDB COMMENT='名句配置';
```

**API 设计**：
| 方法 | 路径 | 说明 |
|:---|:---|:---|
| GET | `/api/quotes` | 获取全部句子（分页可选） |
| POST | `/api/quotes` | 新增句子（支持批量） |
| PUT | `/api/quotes/:id` | 编辑单条句子 |
| DELETE | `/api/quotes` | 批量删除（body: { ids: [] }） |
| GET | `/api/quotes/config/:ruleId` | 获取某规则的名句配置+候选队列 |
| PUT | `/api/quotes/config/:ruleId` | 更新名句配置（enabled, no_repeat_count） |
| POST | `/api/quotes/config/:ruleId/refresh` | 更换一批候选队列 |
| POST | `/api/quotes/config/:ruleId/next` | 使用下一句（跳过当前） |

**候选队列生成算法**：
1. 排除 `used_history` 中最近 N 条已用 quote_id
2. 从剩余句子中随机抽取 10 条，组成 candidate_queue
3. 每次发送值班通知时，取 candidate_queue[0] 作为开始句、candidate_queue[1] 作为结束句
4. 发送后将已用句子追加到 used_history，从 candidate_queue 头部移除，末尾补充新随机句

#### Phase 2：后端 — AutoTaskService 集成

**修改文件**：`backend/src/services/AutoTaskService.js`

**改动内容**：
- 在 `sendDutyEvent()` 发送值班通知前，检查该 rule 是否开启名句搭配
- 若开启：从 quote_config 获取 candidate_queue 第一条(start)/第二条(end)
- 拼接内容：`句子\n原始message`
- 发送后更新 candidate_queue 和 used_history

#### Phase 3：前端 — SettingsPage 名句搭配 UI

**修改文件**：`frontend/src/views/SettingsPage.vue`

**改动位置**：值班通知规则区域 → dt-duty-card 附近（值班概览卡片谭庄左侧值班对象下方）

**新增 UI 元素**：
1. **开关行**：`el-switch` + "开启名句搭配" 文字
2. **去重配置**：`近 [el-input-number] 次句子不得重复`
3. **候选队列面板**：
   - 显示 10 条候选句子，第一条用边框/高亮着重
   - "更换一批" 按钮 + "使用下一句" 按钮
4. **查看全部句子按钮**：打开弹窗

**新增弹窗**（查看全部句子）：
- 按顺序列表显示所有句子
- 最近一次要被使用的句子着重高亮
- 顶部"添加句子"入口（支持输入+确认）
- 每行：序号 + 内容 + 编辑按钮 + 删除勾选框
- 底部：多选批量删除按钮
- 支持内联编辑（点击编辑进入输入模式）

#### Phase 4：前端 — 默认句子数据初始化

**方式**：后端 `/api/quotes` 提供 POST 批量接口，前端或后端初始化脚本一次性导入约 300 条默认句子。

**新建迁移脚本**：`backend/scripts/seed_quotes.js` — 读取默认句子列表，批量写入 quotes 表。

#### Phase 5：测试与归档

- 本地联调验证（不发布生产）
- 更新 CHANGELOG.md

## 注意事项

- 本次优化**不发布生产**
- 名句随机算法需保证：每次值班开始和结束取不同句子
- 近 N 次去重：基于 used_history 数组长度控制
- 候选队列长度固定 10 条，消耗后自动补充
- 默认句子约 300 条，按用户提供原文逐句分割（以句号为间隔，每两个逗号分隔的短句组成一个完整句子）
