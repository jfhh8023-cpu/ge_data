# 导入导出功能 — 需求 & 开发计划

## 一、需求总览

在现有 DevTracker 系统的 4 个页面上新增 Excel 导入/导出能力，所有导入导出均使用 `.xlsx` 格式，上传的 Excel 文件保存到数据库。

| # | 页面 | 新增功能 | 按钮位置 |
|---|------|---------|---------|
| 1 | 填写工时页 (FillPage) | ① 左栏表单新增**导入**按钮 + **下载导入模板** ② 右栏"我的历史"新增**导出**按钮，导出全年数据（按季度+周期分割） | ① 底部操作栏"暂存草稿"左侧 ② 历史面板标题栏右侧 |
| 2 | 任务收集 → 任务详情 (TaskDetail) | "提交数据"Tab 新增 ① **导入**按钮 + **下载导入模板** ② 导入时校验格式，不匹配提示"页面数据格式不匹配，请重新导入" | 刷新按钮左侧 |
| 3 | 需求工时统计 (ReportPage) | ① **导出**按钮：导出当前选中周期的页面数据 ② **导入**按钮 + **下载导入模板**：导入时若已有相同需求名称/版本号则覆盖；导入行可编辑/修改/删除（同手动新增行）③ 导入格式校验 | ① 导出：编辑按钮左侧 ② 导入+模板：导出按钮左侧 |
| 4 | 周期统计 (StatsPage) | **导出**按钮：一模一样导出页面全部内容（概要卡片 + 柱状图PNG + 明细表），Excel 方式 | 刷新按钮左侧 |

## 二、技术选型

| 层级 | 技术方案 | 说明 |
|------|---------|------|
| 前端 Excel 读写 | **xlsx (SheetJS)** | `npm i xlsx`，前端纯浏览器解析/生成 xlsx，无需后端中转 |
| 前端图表导出 | **canvas.toBlob()** | StatsPage 柱状图已用 Canvas，直接转 PNG 写入 Excel |
| 后端 Excel 存储 | 新建 `ExcelFile` 模型 | 存储导入的原始 Excel 二进制 + 元数据 |
| 后端 Excel 解析 | **xlsx (SheetJS)** | `npm i xlsx`，后端也需解析以做服务端校验 |
| 文件上传 | **multer** | `npm i multer`，处理 multipart/form-data |

## 三、数据库变更

### 3.1 新增模型：`ExcelFile`

```sql
CREATE TABLE excel_files (
  id          VARCHAR(36) PRIMARY KEY,
  filename    VARCHAR(255) NOT NULL,        -- 原始文件名
  file_data   LONGBLOB NOT NULL,            -- xlsx 二进制内容
  file_size   INT NOT NULL,                 -- 文件大小(字节)
  source_page VARCHAR(50) NOT NULL,         -- 来源页面: fill / task_detail / report / stats
  task_id     VARCHAR(36),                  -- 关联的任务ID(可为null)
  staff_id    VARCHAR(36),                  -- 关联的人员ID(可为null，FillPage导入时)
  upload_type ENUM('import','export') NOT NULL, -- 上传类型
  created_at  DATETIME DEFAULT CURRENT_TIMESTAMP,
  updated_at  DATETIME DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP
);
```

> 不影响现有任何表结构，纯新增。

## 四、各页面详细方案

### 4.1 填写工时页 (FillPage.vue)

#### 4.1.1 导入功能

- **按钮**: 底部操作栏，"暂存草稿"按钮**左侧**添加 `📥 导入` 按钮 + `📋 下载模板` 链接
- **模板格式** (WorkRecord 维度):

| 需求标题 | 版本号 | 产品经理 | 工时(小时) |
|---------|--------|---------|-----------|
| 示例需求 | V4.633.0 | 钟冠 | 8 |

- **导入逻辑**:
  1. 前端用 SheetJS 解析 xlsx
  2. 校验列头是否匹配 4 列（需求标题/版本号/产品经理/工时）
  3. 格式不匹配 → `ElMessage.error('页面数据格式不匹配，请重新导入')`
  4. 解析成功 → 追加到当前 `rows` 中（若当前仅有一行空行则替换）
  5. 上传原始 xlsx 到后端 `POST /api/excel/upload` 存入 `ExcelFile` 表
  6. 用户仍需手动点"提交"才写入数据库

#### 4.1.2 历史导出

- **按钮**: 右栏"📋 我的历史"标题右侧新增 `📤 导出` 按钮
- **导出逻辑**:
  1. 调 `/api/fill/:token/history` 获取全部历史数据
  2. 按**季度**分 Sheet（Sheet 名: `Q1`, `Q2`, `Q3`, `Q4`）
  3. 每个 Sheet 内按**周期任务**分区（任务标题作为合并单元格标题行）
  4. 每条 WorkRecord 一行：序号 | 需求标题 | 版本号 | 产品经理 | 工时(小时)
  5. 每个任务区末尾有小计行
  6. 前端直接用 SheetJS 生成 xlsx 并触发下载
  7. 同时上传到后端 `POST /api/excel/upload` 存档

### 4.2 任务收集 → 任务详情 (TaskDetail.vue)

#### 4.2.1 导入功能

- **按钮**: 页面标题区刷新按钮**左侧**添加 `📥 导入` + `📋 下载模板`
- **模板格式** (WorkRecord 维度):

| 人员姓名 | 需求标题 | 版本号 | 产品经理 | 工时(小时) |
|---------|---------|--------|---------|-----------|
| 邬涛 | 示例需求 | V4.633.0 | 钟冠 | 16 |

- **导入逻辑**:
  1. 前端解析 xlsx → 校验列头 5 列
  2. 格式不匹配 → `ElMessage.error('页面数据格式不匹配，请重新导入')`
  3. 校验通过 → 发送到后端 `POST /api/records/import`
  4. 后端校验人员姓名是否在 Staff 表中，不在则报错提示
  5. 写入 WorkRecord + 自动触发 MatchGroup 重新匹配
  6. 原始 xlsx 存入 `ExcelFile` 表

#### 4.2.2 导出导入模板

- 前端用 SheetJS 生成含表头 + 1 行示例数据的模板 xlsx
- 或后端 `GET /api/excel/template/:page` 返回预制模板

### 4.3 需求工时统计 (ReportPage.vue)

#### 4.3.1 导出

- **按钮**: 工具条右侧，编辑按钮**左侧**添加 `📤 导出`
- **导出逻辑**:
  1. 导出当前选中周期下 `sortedGroups` 的全部数据
  2. Excel 列：序号 | 版本号 | 需求名称 | 产品经理 | 前端姓名 | 前端工时 | 后端姓名 | 后端工时 | 测试姓名 | 测试工时 | 总计 | 备注
  3. 末行合计行
  4. 前端 SheetJS 生成 + 下载 + 存档到后端

#### 4.3.2 导入

- **按钮**: 导出按钮**左侧**添加 `📥 导入` + `📋 下载模板`
- **模板格式** (MatchGroup 维度):

| 版本号 | 需求名称 | 产品经理 | 前端姓名 | 前端工时 | 后端姓名 | 后端工时 | 测试姓名 | 测试工时 | 备注 |
|--------|---------|---------|---------|---------|---------|---------|---------|---------|------|

- **导入逻辑**:
  1. 前端解析 xlsx → 校验列头 10 列
  2. 格式不匹配 → `ElMessage.error('页面数据格式不匹配，请重新导入')`
  3. 发送到后端 `POST /api/report/import?taskId=xxx`
  4. **覆盖规则**: 若已存在**相同需求名称**或**相同版本号**的 MatchGroup，则**覆盖**保存（更新现有记录）
  5. 新行 status 设为 `manual_merged`（可编辑/修改/删除，与手动新增行一致）
  6. 同时同步创建对应 WorkRecord
  7. 原始 xlsx 存入 `ExcelFile` 表

## 九、格式校验规则汇总

每个导入入口统一执行：

1. **文件类型校验**: 仅接受 `.xlsx` / `.xls`
2. **列头校验**: 解析第一行，与预期列名逐一比对（忽略前后空格）
3. **数据行校验**: 跳过全空行；工时列须为数字且 > 0；人员姓名须在系统 Staff 表中存在（TaskDetail 导入时后端校验）
4. **不匹配处理**: 弹出 `ElMessage.error('页面数据格式不匹配，请重新导入')`，不做任何数据写入

## 十、不影响现有功能保证

- 所有按钮为**纯新增**，不修改现有按钮和交互逻辑
- Excel 存储使用**独立新表** `ExcelFile`，不影响现有表结构
- 导入数据走现有 API 逻辑（WorkRecord.create / MatchGroup.create），确保数据一致性
- ReportPage 导入的行 status 为 `manual_merged`，与现有手动新增行行为完全一致
- 所有后端新增路由挂载在独立 `/api/excel` + 已有路由的新子路径上
